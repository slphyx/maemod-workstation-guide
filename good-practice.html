Good Practice

This page gives recommended practices for using the shared workstation.

1 Use Slurm for long jobs

Do not run long or intensive jobs directly in the login terminal.

Use Slurm:

sbatch job.sh

2 Organise project folders

Recommended structure:

project-name/
├── data/
├── scripts/
├── results/
├── figures/
├── reports/
└── README.md

3 Keep raw data unchanged

Do not edit raw data directly.

Save cleaned or transformed data in a separate folder.

4 Use clear filenames

Good examples:

analysis-model-v1.R
results-2026-06-18.csv
figure-transmission-map.png

Avoid:

new.R
test2.csv
final_final_v3.csv

5 Record software versions

Useful commands:

R --version
python --version
quarto --version

For R:

sessionInfo()

For Python:

pip freeze

or:

conda list

6 Use Git for code

Use Git for scripts, reports, and documentation.

Do not put very large data files into Git unless there is a clear reason.

7 Clean up old files

Check folder size:

du -sh .

Remove temporary files that are no longer needed.

8 Be considerate

This is a shared workstation.

Please avoid using all CPUs or memory unless necessary.
